# RHCSA-Linux用户和权限管理

## 用户和用户组的概念

### 用户有什么作用?

- 登录系统是需要用户的
- 系统上运行的每个应用程序, 都需要用户来进行运行
- 用户还可以进行权限管理

### 在Linux系统中, 用户的分类

- 超级管理员(root用户): 这个是特殊的用户, 是不需要手动创建的用户, 安装Linux系统默认自带, 作为系统超级管理员可以管理系统, 比如修改网络配置文件,修改系统主机名......
- 普通用户: 由超级管理员创建的用户, 一般普通用户是给到用户登录使用的
- 系统用户: 这类用户一般是不可以登录系统的, 虽然不可登录, 但可以用来运行软件程序......

### 系统如何判断用户是属于哪类用户

- 根据用户的UID进行判断的
- UID: User ID 用户的ID, Linux系统中每一个用户都具备一个UID
  - **UID为0**的用户属于**超级管理员**→root用户的UID就是0 **(UID为0的是超级管理员, root用户不一定是超级管理员)**
  - UID的范围在0到1000之内, 属于系统用户
  - UID的范围大于等于1000, 属于普通用户

### 用户组

- 在Linux系统中, 除了有用户的概念, 还有用户组的概念: **用户组就是多个用户的集合, 在一个用户组里面可以包含一个或多个用户**
- 用户组方便管理用户
- 用户组的分类
  - **私有组(属组)**: 一个用户必须要属于一个用户组, 这个用户组称之为用户的私有组
  - **附加组(公共组)**: 一个用户可以加入多个其他的用户组中, 其他的用户组就属于这个用户的附加组
    GID: Group ID 用户组ID, 每一个用户组都会分配一个GID

### 用户信息配置文件

- 系统是无法识别用户名的, 只能识别UID. 通过命令查询用户信息的时候, 会显示用户名和UID的映射关系, 这是因为系统上有个文件/etc/passwd

  ![](image_x2JFnZuo97.png)
  - 第1列(root): **用户的名字**
  - 第2列(x): **密码站位符**, 使用用户登录系统是否验证密码, 如果有x表示需要验证, 没有x表示不需要验证
  - 第3列(0): **用户的ID, UID**
  - 第4列(0): **私有组ID, GID**
  - 第5列(root): **用户的描述信息**, 在图形化界面这个描述信息才有用处
  - 第6列(/root): **用户的家目录**
  - 第7列(/bin/bash): **登录shell程序**, 用户登录系统运行的第一个shell程序(命令, 就是登录系统后, 用户第一个执行的Linux命令)
    - **/bin/bash**: 表示用户可以登录系统
    - **/sbin/nologin**: 表示用户不可以登录系统

### 用户密码文件

- 系统上每一个用户设置密码的时候, 密码其实是经过加密的, 加密后的密码保存到 **/etc/shadow密码文件中**

  ![](image_X9bZ1IY4w3.png)
  - 第1列(root): **用户名**
  - 第2列(\$$6$\$pB9Q1to4ltg.WY1I\$YarWk6k): **加密后的密码**
    - \$6\$: hash512方式加密 (Linux中默认使用hash512方式加密)
    - \$5\$: hash256方式加密
    - $1$: md5方式加密
      当用户使用passwd命令修改用户密码的时候, 默认使用hash512方式对你输入的密码进行加密, 然后将加密后的字符串写入到 **/etc/shadow**文件中
    使用**openssl passwd -6**命令对字符串进行加密
  - 第3列(20771): **上一次修改密码的时间**,\*\* 20771是天数\*\*→从**1970-01-01**开始计算,经过了**20771天**
  - 第4列(0): **密码最短有效期**, 两次密码修改之间的最短期限, 必须要超过这个期限才能更改密码, 如果是**0的话表示无限制**, 单位是天数, **超级管理员用户没有这个限制**
  - 第5列(99999): **密码最长有效期**, 密码最长能够使用多长时间, 如果超过了这个时间, 再次登录的时候系统会强制要求你修改用户密码. 如果是**99999**, 表示无限制, 单位是天数
  - 第6列(7): **密码到期前多少天进行告警**, 看的是密码最长有效期
  - 第7列: **密码过期后多少天会锁定账号**, 锁定后登录的话是无法登录的, 需要联系超管解锁账户
  - 第8列: **密码锁定日期**, 单位是天数, 指定哪一天直接账户过期
  - 第9列: 保持为空

### 用户管理命令

- 创建用户: useradd
- 查看用户: id
- 删除用户: userdel
- 修改用户: usermod

#### useradd创建用户

- 语法格式
  ```bash 
  用法：useradd [选项] 登录名  #最常用
        useradd -D
        useradd -D [选项]

  ```

- 创建mubai1用户
  ```bash 
  [root@mubai632 ~]# useradd mubai1
  [root@mubai632 ~]# id mubai1
  用户id=1001(mubai1) 组id=1001(mubai1) 组=1001(mubai1)

  ```

  - 用户创建的时候没有指定其Id号, 系统自动分配一个可用的id号
  - 分配原则:
    - UID是从1000开始进行分配
    - 新的普通用户的UID是/etc/passswd文件中UID最大的数+1
      **并且, 创建用户的同时还会创建一个和用户名相同的用户组, 作为这个用户的私有组**
  创建UID为0的用户, 会报UID不唯一, 但是还是可以创建的, 使用 -o选项(不常用)
- 常用选项
  ```bash 
  useradd -u 指定uid
  useradd -o 复用uid
  useradd -g 指定私有组
  useradd -G 指定附加组
  useradd -c 指定描述信息，描述信息最好用引号包裹
  useradd -d 指定家目录，默认会自动创建家目录
  useradd -s 指定登录shell
  useradd -M 不创建家目录
  useradd -p 指定密码, 但是需要使用 openssl passwd 命令进行加密, 并且 -p 后面需要用单引号包裹
  ```

- 例子
  ```markdown 
  #创建名为test用户，要求UID为3333，描述信息为test user，家目录在/opt/test，登录shell为/bin/bash
  [root@mubai632 ~]# useradd -u 3333 -c 'test user' -d /opt/test -s /bin/bash test

  ```


#### userdel删除用户

- 语法格式
  ```bash 
  用法：userdel [选项] 登录名
   常用选项: -r , --remove , 递归删除, 删除用户的同时会删除家目录和邮箱文件
            -Z , --selinux-user , 为用户删除所有的 SELinux 用户映射

  ```

- 删除用户的前提是: 用户未登录

  ![](image_n-cERiRkIy.png)
  - 出现这种报错的原因是:&#x20;
    1. 其他的地方登录了demo用户, 远程或者是终端
    2. 你从dome用户登录到了其他用户, 你最初使用的登录用户是dome
- **如果不使用 -r 选项, 删除时会提示警告信息, 提示你家目录和邮件文件未删除, 但是会删除成功, 这仅仅只删除了用户而已**
- **删除后创建同名的用户, 虽然UID还是沿用新建用户的原则(最大UID +1), 但会提示警告信息, 用户的家目录和邮箱文件存在, 导致无法创建这两个文件,  并且文件的家目录的权限会是原用户的UID, 导致新创建的用户没有可用的家目录**

#### usermod修改用户

- usermod修改的是已经存在的用户信息
  - 修改用户的名称,用户的UID,用户的附加组,用户的描述信息,
- 语法格式
  ```markdown 
  用法：usermod [选项] 登录名



  ```

- 常用选项
  ```bash 
  usermod -l , --login : 修改用户名
  usermod -u , --uid : 修改用户的ID, UID
  usermod -g , --gid : 修改GID
  usermod -G , --groups : 将用户添加到一个或多个附加组中(用户无附加组使用)
  usermod -aG : 将用户添加到一个或多个附加组中(用户有一个或多个附加组使用)
    格式是 usermod -aG 组,组... 用户名 (组看用引号包裹起来)
       or usermod -a 用户名 -G 组,组...
  usermod -c , --comment : 修改描述信息
  usermod -s , --shell : 修改登录shell
  usermod -L , --lock : 锁定用户账号
  usermod -U , --unlock : 解锁用户账号
  usermod -md : 移动用户的家目录中的文件, 到新创建的家目录中, 并修改家目录配置文件
    usermod -d : 只会修改/etc/passwd文件中用户的家目录. 但不会创建用户的家目录
    usermod -m : 将家目录内容移至新位置


  ```

- 例子
  1. 将mubai2用户的名字修改为mubai
     ```bash 
     [root@mubai632 ~]# usermod -l mubai632 mubai2
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2)

     ```

  2. 修改mubai632用户的描述信息和登录shell，描述信息为mubai632，登录shell为/sbin/nologin
     ```bash 
     [root@mubai632 ~]# usermod -c "mubai632" -s /sbin/nologin mubai632
     [root@mubai632 ~]# cat /etc/passwd | grep mubai632
     mubai632:x:3334:3334:mubai632:/home/mubai2:/sbin/nologin

     ```

  3. 将mubai632用户登录shell修改为/bin/bash，锁定mubai632用户，然后尝试登录，最后进行解锁mubai632用户
     ```bash 
     [root@mubai632 ~]# usermod -s /bin/bash mubai632
     [root@mubai632 ~]# cat /etc/passwd | grep mubai632
     mubai632:x:3334:3334:mubai632:/home/mubai2:/bin/bash

     [root@mubai632 ~]# usermod -L mubai632
     [mubai@mubai632 ~]$ su - mubai632
     密码：
     su: 鉴定故障
     [root@mubai632 ~]# cat /etc/shadow | grep mubai632
     mubai632: ! $6$Gwm/NCZb42UZrpio$OrS7yozYVDHORia8QdtYO34.l1Dlg8XDnpNbBKoJrPIb7Twucdt0E.KWmnGVwywZpTC6dbEEXhnDejJACjyG81:20489:0:99999:7:::
     #将加密的密码前面加一个  !  , 验证会失败

     [root@mubai632 ~]# usermod -U mubai632
     [mubai@mubai632 ~]$ su - mubai632
     密码：
     [mubai632@mubai632 ~]$
     [root@mubai632 ~]# cat /etc/shadow | grep mubai632
     mubai632:$6$Gwm/NCZb42UZrpio$OrS7yozYVDHORia8QdtYO34.l1Dlg8XDnpNbBKoJrPIb7Twucdt0E.KWmnGVwywZpTC6dbEEXhnDejJACjyG81:20489:0:99999:7:::
     #密码前面的 ! 去掉就是解锁账号, 验证可以通过

     ```

  4. 将mubai632用户加入到root组中, 再移除附加组
     ```bash
     [root@mubai632 ~]# usermod -G root mubai632
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root)
     #如果用户已有一个以上的附加组, 就要添加 -a 参数进行追加
     [root@mubai632 ~]# usermod -a mubai632 -G mubai
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root),1000(mubai)
     #也可以写成 -aG 进行追加, 选项参数是 追加的组 用户
     [root@mubai632 ~]# usermod -aG mubai1 mubai632
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root),1000(mubai),1001(mubai1)

     #一次性对一个用户添加多个组
     #方式1
     [root@mubai632 ~]# usermod -aG mubai1,mubai,root  mubai632
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root),1000(mubai),1001(mubai1)
     #方式2(单引号双引号都可以)
     [root@mubai632 ~]# usermod -aG 'mubai1,mubai,root'  mubai632
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root),1000(mubai),1001(mubai1)
     #方式3( 在用户没有附加组的情况下 )
     [root@mubai632 ~]# usermod -G mubai1,mubai,root  mubai632
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root),1000(mubai),1001(mubai1)
     #方式4( 在用户没有附加组的情况下 , 单引号双引号都可以)
     [root@mubai632 ~]# usermod -G 'mubai1,mubai,root'  mubai632
     [root@mubai632 ~]# id mubai632
     用户id=3334(mubai632) 组id=3334(mubai2) 组=3334(mubai2),0(root),1000(mubai),1001(mubai1)

     #移除用户全部附加组, 如果想保留附加组, 将想保留的附加组替换掉 '', 如果是多个使用 , 隔开 
     [root@mubai632 ~]# usermod -G '' mubai632
     ```


### 用户密码管理命令

passwd命令修改用户的密码

chage命令修改用户密码相关信息→/etc/shadow文件内容

- 语法格式
  ```bash 
  用法：chage [选项] 登录名
  常用选项:
    -I, --inactive n : 过期n天锁定账号
    -l, --list : 显示用户时间信息
    -m, --mindays 最小天数 : 密码最短有效期
    -M, --maxdays MAX_DAYS : 密码最长有效期
    -E, --expiredate 过期日期 : 账户锁定日期
    -W, --warndays 警告天数 ; 密码失效前多少天告警
    -d, --lastday 最近日期 : 设定密码更新时间

  ```

- 查看用户信息
  ```bash 
  [root@mubai632 ~]# chage -l mubai1
  最近一次密码修改时间(Last password change)         ：2月 04, 2026
  密码过期时间(Password expires)                    ：3月 06, 2026
  密码失效时间(Password inactive)                   ：3月 08, 2026
  帐户过期时间(Account expires)                     ：5月 15, 2026
  两次改变密码之间相距的最小天数(Minimum number of days between password change)    ：3
  两次改变密码之间相距的最大天数(Maximum number of days between password change)    ：30
  在密码过期之前警告的天数(Number of days of warning before password expires)       ：7
  [root@mubai632 ~]# cat /etc/shadow | grep mubai1
  mubai1:!!:20488:3:30:7:2:20588:

  ```

- 强制要求用户登录后修改密码
  ```markdown 
  #将密码更改日期设置为0, 你登录系统就会要求你强制更改密码
  [root@mubai632 ~]# chage -d 0 mubai1
  [root@mubai632 ~]#  chage  -l mubai1
  最近一次密码修改时间                 ： 密码必须更改(password must be changed) 
  密码过期时间                        ： 密码必须更改(password must be changed) 
  密码失效时间                        ： 密码必须更改(password must be changed) 
  帐户过期时间                        ：5月 15, 2026
  两次改变密码之间相距的最小天数          ：3
  两次改变密码之间相距的最大天数          ：30
  在密码过期之前警告的天数        ：7
  ```

- 例子
  - 要求修改mubai3用户的密码相关信息, 此用户密码最短有效期为7天, 最长有效期为30天, 密码过期前5天开始告警, 密码过期后3天进行锁定, 到2027/1/25账户失效

    ```bash 
    两次改变密码之间相距的最大天数          ：30

    ```

### 用户组信息配置

用户组是多个用户的集合, 一个组内的所有的用户都可以共享这个组的权限

- 组分类
  - 私有组
  - 附加组
- **useradd命令创建用户时, 会同时创建一个和用户同名的组, 作为这个用户的私有组**
- **组的信息记录在****/etc/group文件中, 这个文件记录了 组名和GID映射关系,组内成员信息**​

  ![](image_vNMVNPxwta.png)
  - 第1列: 组名
  - 第2列: 密码占位符(如果时x表示这个组需要密码验证才可以登录, 如果没有x表示不需要验证)
    - 给组设置密码, 目的是为了让用户通过密码登录到组
    - 如果用户登录到了某个组, 那么这个组会临时成为这个用户的私有组, 登录到组的目的是为了获得这个组的权限
    - gpasswd命令给组设置密码
    - newgrp命令登录到组中
  - 第3列: GID
  - 第4列: 组内的成员, 成员之间用 , 隔开

### 用户组密码文件

- **/etc/gshadow** : 保存组的密码, 组的管理员, 组的成员

  ![](image_zzsLx3n5sU.png)
  - 第1列: 组名
  - 第2列: 加密后的组密码
  - 第3列: 组的管理员
    - 组的管理员可以对组的成员进行管理, 加入或踢出

- 常用选项
  ```bash

    -a, --add USER                向组 GROUP 中添加用户 USER

    -h, --help                    显示此帮助信息并退出
    -Q, --root CHROOT_DIR         要 chroot 进的目录
    -r, --delete-password         remove the GROUPs password
    -R, --restrict                向其成员限制访问组 GROUP
    -M, --members USER,...        设置组 GROUP 的成员列表(覆盖)
    -A, --administrators ADMIN,...     设置组的管理员列表(覆盖)
  除非使用 -A 或 -M 选项，不能结合使用这些选项。

  #gpasswd不加选项直接接组名，表示设置组的密码；通过newgrp命令可以临时登录到这个组
  ```


## 文件权限

如果系统没有权限限制, 那么所有的用户都可以访问系统的配置文件

文件权限是赋予给文件的

- 文件权限有3种
  - r (读权限): 用户可查看文件内容
  - w (写权限): 用户可编辑文件内容
  - x (可执行权限): 用户可执行这个文件

```markdown 
[mubai632@mubai632 ~]$ ll /usr/bin/ls
- rwx r-x r-x . 1 root root 140760  1月  6  2023 /usr/bin/ls
 rwx: 拥有者对文件的权限 
 r-x: 所属组对文件的权限 
 r-x: 其他人对文件的权限 

#按照顺序进行匹配, 没有匹配上就继续向下走
如果用户的UID和文件的UID一致, 则匹配文件的拥有者权限
如果用户组的GID和文件的GID一致, 则匹配文件的所属组权限
都未匹配, 则以其他人的权限来访问
```


### 文件权限的表达方式

- 字符方式
  - r : 读
  - w : 写
  - x : 执行
- 数字方式
  - r : 4
  - w : 2
  - x : 1

### 文件权限对于文件的作用

- 普通文件
  ```markdown 
  r : 如果一个普通文件具备r权限, 则表示可以读取文件内容
  w : 如果一个普通文件具备w权限, 则表示可以编辑文件内容
  x : 如果一个普通文件几倍x权限, 则表示可以执行文件
  # 使用命令可以查看命令的绝对路径
    which <命令>

  ```

- 目录文件
  ```markdown 
  r : 如果一个目录文件具备r权限, 则表示可以列出目录下的内容, 但无法进入这个目录
  x : 如果一个目录文件具备x权限, 则表示可以进入这个目录, 但无法看到详细信息
  w : 如果一个目录文件具备w权限, 则表示可以在目录下创建和删除文件
  ```

### 文件权限的查看以及修改

- 查看文件权限两种方式
  - ls -l命令
  - stat命令
- chmod命令修改文件权限
  - 基于字符修改语法格式
    ```bash 
    chmod [选项] 对象 属性操作符 权限 文件..
      对象:
        u : 修改拥有者权限
        g : 修改所属组权限
        o : 修改其他人权限
        a : 修改所有人的权限
      属性操作符: 
        + 在原有权限基础上新增权限
        - 在原有权限基础上减少权限
        = 直接赋予给定权限
      权限: 
        r, w, x
      常用选项: 
        -R : 递归赋予权限, 将目录下的全部文件都赋予对应权限

    ```

  - 基于数字修改语法格式
    ```bash 
    chmod xxx 文件
      r : 4
      w : 2
      x : 1

    ```

- 例子
  - 要求给/etc/passwd文件修改文件权限, 拥有者只读, 拥有组读写, 其他人没有权限
    ```bash 
    [root@mubai632 ~]# chmod u-wx,g-x,g+w,o-rwx /etc/passwd
    [root@mubai632 ~]# ll /etc/passwd
    -r--rw----. 1 root root 2183  2月 20 10:01 /etc/passwd
    ```


### 修改文件的拥有者和所属组

![](image_ZMxwE2xKCX.png)

- 要求如下：
  - 不能够通过chmod修改文件权限，要求某一个用户可以对/etc/passwd文件进行读写操作
  - 当前的解决方案只有一个：就是将/etc/passwd文件的拥有人修改为该用户
- chown : 修改文件的拥有者和所属组
  - 语法格式:
    ```bash 
     chown [选项]... 拥有者:所属组 文件...
     常用选项: -R : 递归更改
    ```

  - 例子
    ```markdown 
    # 只修改文件的拥有者
    [root@mubai632 test]# chown mu /etc/test/123
    [root@mubai632 test]# ll
    -rw-r--r--. 1 mu root 0  2月 21 16:07 123

    # 只修改文件的所属组
    [root@mubai632 test]# chown :mu /etc/test/123
    [root@mubai632 test]# ll
    -rw-r--r--. 1 mu mu 0  2月 21 16:07 123
    或者
    [root@mubai632 test]# chown .mu /etc/test/123
    [root@mubai632 test]# ll
    -rw-r--r--. 1 mu mu 0  2月 21 16:07 123

    # 更改文件的拥有者和所属组
    [root@mubai632 test]# chown mfb:mfb /etc/test/123
    [root@mubai632 test]# ll /etc/test/123
    -rw-r--r--. 1 mfb mfb 0  2月 21 16:07 /etc/test/123
    或者
    [root@mubai632 test]# chown mfb.mfb /etc/test/123
    [root@mubai632 test]# ll /etc/test/123
    -rw-r--r--. 1 mfb mfb 0  2月 21 16:07 /etc/test/123

    ```

- chgrp : 只能够修改文件的所属组
  - 语法格式
    ```bash 
    chgrp [选项]... 用户组 文件...

    [root@mubai632 test]# chgrp mfb  /etc/test/123
    [root@mubai632 test]# ll /etc/test/123
    -rw-r--r--. 1 mu mfb 0  2月 21 16:07 /etc/test/123

    ```


作为超管来说，也就是root用户。无论是不是自己的文件，都可以修改文件的拥有人和拥有组身份

对于普通用户来说，文件的拥有人只能够将文件的拥有组修改自己的
