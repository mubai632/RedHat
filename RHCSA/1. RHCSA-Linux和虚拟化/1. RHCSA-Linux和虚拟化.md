# RHCSA-Linux和虚拟化
### Linux应用场景

- 什么是LInux?
  - linux是一个开源的操作系统, 支持多用户多进程
  - Linux其实是一个内核而已, 通过内核+应用形成了操作系统
- Linux应用
  - 虚拟化和云平台
  - 大数据
  - 分布式软件
  - 嵌入式Ubuntu系统

### 开源软件和开源协议

- 什么是开源软件\开源协议?
  - 开源就是将源代码进行公开, 任何人都能看见
- 自由基金会定义了四大自由
  - 允许以任何目的运行软件
  - 允许学习和修改软件源代码
  - 允许重新发布软件
  - 允许开发软件的衍生版本
- 常见开源许可证
  - GNU GPL(GNU通用公共许可证): 基于这个开发的都需要开源
  - BSD(伯克利软件发布版)
  - Apache许可证版本: 必须说明你用了那个开源软件的哪些代码
  - MIT许可协议

- UNIX→GNU→Minix→Linux
- Linux内核版本: x.xx.x xx为偶数为稳定版本, xx为奇数就是开发版本

### 红帽认证体系和发展前景

- Linux发行版本很多, 但是只有两个方向:
  - 基于RHEL: RHEL, Fedora, CentOS, OpenEuler......
  - 基于Debian: Debian, Ubuntu....
    这两个版本最大的区别就是包管理器的不同:&#x20;
  - RHEL系列是rpm, yum或者dnf
  - Debian系列都是deb或者apt

      - test mode安装操作系统, 命令行安装
      - 拯救红帽操作系统(救援模式), Linux故障修复
      - 检查内存情况
      - 从硬盘中加载启动
      - 返回主菜单

  - 设置网络, 需要将常规(General)中的(Connect Automatically with priority)勾选上, 要不然系统启动不会自动连接网络

  - 创建分区: 蓝色字体自动创建分区(但是不使用), 打开分区方案
    - Standard Partition(标准分区): 前期使用
    - LVM(LVM分区): 逻辑卷
    - LVM Thin Provisioning(LVM简单配置): 精简模式的逻辑卷
  - 挂载点: 磁盘划分出一个区域, 挂载到/boot分区下, /boot称之挂载点
    ![](image_AUJqyI6F9D.png)
  - 对于Linux系统最重要且必须的2个分区, (/boot)分区和根(/)分区
    - **/boot分区: 保存Linux内核文件, 以及Linux启动时加载的文件的分区**
    - **/ 分区: 根分区, 重要性相当于windows C盘**
    - /home分区: 保存Linux系统普通用户的家目录的
    - /var分区: 保存变化的数据
    - swap分区: swap就是将硬盘的一部分空间当做内存使用. 软件只能运行在内存中, 当内存不够时, swap分区就会充当内存
  - 安全配置文件生产和学习都不要打开, 它会强制要求你划分不必要的分区
  - 用户设置
    - 设置root用户, RHEL9会有两个选项, 一个是锁定root用户, 一个是允许远程ssh登录, 一般情况这两个都不选
    - 还有创建用户, 需要你创建一个普通用户

### 虚拟化基础

- 物理机安装Linux系统
  1. 准备一个U盘或者光驱：U盘里面得搞一个Linux的安装介质，做一个Linux的启动盘
     1. 启动盘的制作方式：准备一个U盘，准备一个ISO镜像文件，准备一个封装启动盘的软件rufus
     2. 得到了一个启动盘，插入到物理服务器的USB接口，识别U盘
     3. 重启物理服务器，开机的时候选择从U盘启动—>F12 boot，选择启动项
     4. 启动项：硬盘/U盘（光驱）/网络启动
  2. 通过物理服务器的管理口进行登录, 登录到虚拟控制台使用ios文件进行安装, 或者使用KVM连接工具连接到虚拟控制台(需要在服务器面前)
  3. PXE批量安装: 通过网络方式安装炒作系统
     1. 一次给多台服务器进行安装
- 如何重新安装Linux操作系统
  1. 选择你要重新安装的操作系统的镜像文件
  2. 选择使用光盘启动
     1. 临时指定启动项: 再Vmware中, 开机使用esc键, 就会跳转选择启动项
     2. 永久指定启动项: 更改BIOS中的默认启动项, 更改完后, 按F10保存退出
- 虚拟化是什么
  - 虚拟机: 通过虚拟化技术, 虚拟出一个主机
  - 实体的物理电脑是宿主机, 运行在宿主机上的vmware虚拟机 就是客户机, 其实就是一堆文件
  - 当物理机存在Hypervisor VMM虚拟化层, 虚拟化监视器. 你的电脑才能创建虚拟机, 才具备虚拟化的能力
  - 虚拟化技术: KVM虚拟化, 基于内核的虚拟化
  - 虚拟化的类型:&#x20;
    1. 一型虚拟化:  裸金属虚拟化, 操作系统就是Hypervisor层, 也就是虚拟化层(共3层 KVM)
    2. 二型虚拟化: 寄居型虚拟化(共4层, vmware)
