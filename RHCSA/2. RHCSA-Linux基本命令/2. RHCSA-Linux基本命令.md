#### 创建用户
- 只有超级管理员才有权限创建用户, 普通用户不能创建用户
```bash 
#创建用户mfb
[root@mubai632 ~]# useradd mfb

#查看创建的用户
[root@mubai632 ~]# id mfb
uid=1001(mfb) gid=1001(mfb) groups=1001(mfb)

#设置新创建的用户的密码
[root@mubai632 ~]# passwd mfb
Changing password for user mfb.
New password:
BAD PASSWORD: The password is shorter than 8 characters
Retype new password:
passwd: all authentication tokens updated successfully.
#第一次输入密码报错, 是因为第一次输入的密码不满足系统的要求, 但是系统还是让你再输入一次, 并且成功创建, 原因是因为超级管理员用户是可以无视系统要求的密码规则

#通过命令登录新创建的用户
[root@mubai632 ~]# su - mfb
[mfb@mubai632 ~]$
[root@mubai632 ~]# su mfb
[mfb@mubai632 root]$
#切换用户一定要使用 su - 用户名 的方式进行切换

```
- passwd命令
  - passwd命令，后面什么都不写，默认是给当前用户修改密码
  - passwd命令，要给指定用户修改密码，后面接上用户名字。只有root用户才可以接用户名
  - 超级管理员修改任何用户密码的时候都不需要验证用户的原有密码, 普通用户修改自己的密码的时候，是需要验证当前用户密码的, 普通用户修改密码的时候，必须要遵循密码策略！
- root用户修改密码：
  可以给任何用户修改密码，并且不需要验证用户的原有密码是什么，也不需要遵循密码策略
- 普通用户修改密码：
  只能够给自己修改密码，需要验证用户的原有密码，并且需要遵循密码策略
- root用户切换其他用户身份的时候，不需要验证用户密码
- 普通用户切换其他用户身份的时候，需要验证用户密码

### Linux三要素
- 命令: 实现具体的功能
- 选项: 对命令功能的补充
  - 辨别选项的方式: 只要你看到是-或者是- -开头的, 都是属于命令的选项
  - 对于选项来说, 段格式的选项是可以连着写的, 长格式选项不行
- 参数: 可以是一切对象, 可以是文件,目录,用户名......

```bash 
[mubai632@mubai632 ~]$ ls -l /home/mubai632/

```
- 语法格式:&#x20;
  - 命令和选项之间必须空格隔开, 多少个空格数量无所谓  (ls -l)
  - 选项和选项之间必须空格隔开, 多少个空格数量无所谓 (ls -l -a)
  - 选项和参数之间必须空格隔开, 多少个空格数量无所谓  (ls -l -a /root/xym)
  - 参数和参数之间必须空格隔开, 多少个空格数量无所谓  (ls -l -a /root/xym /etc/passwd)
